## 1. 목표 재정의

현재 솔버는 기능적으로는 충분히 강력하지만, 시험 환경에서의 **신뢰성**이 부족함.

### 핵심 목표
- 기능 확장이 아닌 **오답 최소화 중심 설계**
- 특히 **T/F 문제에서의 False 판별 정확도 개선**

### 핵심 지표
- 전체 정확도
- False 문항 정확도
- 자동 판정 가능 비율
- 불확실 판단 회피율 (보류 전략)

## 2. ConceptDB 개선 (최우선)

현재 문제:
- 키워드 매칭 기반 → 시험형 함정 문제에 취약

### 2.1 구조 개선

기존: rule = 키워드

개선:

- topic
- claim
- 조건 (valid_for / invalid_for)
- 수치 조건
- 적용 범위


### 2.2 문장 파싱 단계 추가

문장을 다음 요소로 분해:

- 핵심 주장
- 조건
- 수치
- 범위 표현 (only, both, always 등)
- 부정어 (not, cannot)


### 2.3 판단 로직 재구성

기존:
- 가장 유사한 rule → verdict 반환

개선:

1. 유사 rule 탐색
2. 문장 구조 비교
3. 충돌 검사 (조건/수치/부정)
4. 최종 판단


### 2.4 핵심 보완 로직

- 부정어 처리
- 차원 비교 (N vs 2N)
- 적용 범위 비교
- 조건 추가 감지


### 2.5 운영 전략

단계적 적용:

Phase 1:
- 자동 정답 출력 금지
- 개념 + 주의 포인트 출력

Phase 2:
- False 정확도 기준 충족 시 자동화

기준:
- 전체 정확도 ≥ 85%
- False 정확도 ≥ 75%

## 3. 문제 유형 기반 구조로 전환

현재: 기능 모듈 중심

문제: 사용자가 직접 기능 선택 필요

### 개선: 문제 유형 라우터

입력 → 자동 분류 → 적절한 엔진 호출


### 유형 분류 예

- 3차 ODE + FRF → FRF 모듈
- equilibrium + stability → Lagrange 모듈
- eigenvalue → Modal 모듈
- damping → Damping 모듈
- gyroscopic → Extended 모듈


### 목표

"문제 입력 → 자동 풀이 경로 선택"
